# Agent Guidelines — Owner Housing

## Project Context

Static property listing site for India. TanStack Start + SSG. No backend, no auth. Data in `src/data/properties.json`. Vanilla CSS design system in `src/styles.css`.

## Hard Rules

1. **No backend code.** This is a static site. No APIs, no databases, no server functions.
2. **No authentication.** No login, signup, or user accounts.
3. **No Tailwind utility classes in components.** Use semantic CSS classes from `src/styles.css`. Tailwind is imported only for its reset.
4. **No new dependencies** without explicit user approval.
5. **Preserve SEO.** Every page must have: unique `<title>`, meta description, OG tags. Detail pages must have JSON-LD (`RealEstateListing` + `BreadcrumbList`).
6. **Indian locale only.** Prices in ₹ (lakhs/crores). Cities are Indian. Phone numbers are Indian (+91).
7. **Keep `llms-full.txt` in sync.** After changing `properties.json`, run `node scripts/generate-llms-full.mjs` or `npm run build`.

## Architecture

```
src/data/properties.json  → Single source of truth
src/utils/formatPrice.ts  → Price formatting, types
src/utils/seo.ts          → JSON-LD + meta generators
src/components/*          → Reusable UI (no route logic)
src/routes/*              → Pages (use components, call seo utils)
src/styles.css            → All styles (BEM-like naming)
public/                   → Static assets (robots.txt, sitemap, llms.txt)
scripts/                  → Build-time generators
```

## When Adding a Listing

1. Add entry to `src/data/properties.json`
2. Add URL to `public/sitemap.xml`
3. Run `npm run build` (auto-generates `llms-full.txt`, prerender discovers new page)

## When Adding a Component

1. Create in `src/components/`
2. Add styles to `src/styles.css` using BEM naming (`block__element--modifier`)
3. No inline styles. No Tailwind classes.

## Don't Touch

- `src/routeTree.gen.ts` — auto-generated by TanStack Router
- `node_modules/`
- `dist/` — build output
